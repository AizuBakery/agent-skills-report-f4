# 微信消息桥接器

轮询监听微信聊天中的指令消息，自动触发对应的 OpenClaw skill 执行。

## 启动

```bash
# 后台监听"文件传输助手"，每30秒检查一次
~/.openclaw/skills/wechat-bridge/scripts/start.sh "文件传输助手" 30

# 单次检查
~/.openclaw/skills/wechat-bridge/scripts/check.sh "文件传输助手"
```

## 支持的指令

在微信聊天中发送以下文字即可触发：

| 指令 | 效果 |
|------|------|
| `处理图片` | 运行完整 campaign 流水线（清理+风格+对比图+GIF+推文） |
| `处理图片 xxx.jpg` | 处理指定图片 |
| `生成推文` | 从已有对比图生成推文文案和GIF |
| `生成推文 场景描述` | 带场景描述生成推文 |
| `生成图片 描述` | AI 文生图 |
| `修图 描述` | 编辑 pending 目录中最新一张图 |
| `发公众号 标题` | 触发公众号发布流程 |
| `状态` | 查看系统状态 |

## 工作原理

1. 每隔 N 秒截图微信聊天窗口 + OCR 提取文字
2. 检测右侧（自己发的）最新消息是否匹配指令
3. 匹配则调用对应 Python 脚本执行
4. 执行结果通过微信回复

## 注意事项

- 桥接器监听的是**你自己发的消息**（右侧），不是别人发的
- 建议用"文件传输助手"作为指令通道，避免干扰正常聊天
- 状态文件：`/tmp/wechat_bridge_state.json`
- 日志文件：`/tmp/wechat_bridge.log`
